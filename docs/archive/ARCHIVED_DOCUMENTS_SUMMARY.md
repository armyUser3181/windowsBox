## 주요 설계 결정 사항 요약

### 채택된 설계 안

1. **EventRepository 중앙 관리**: EventCaller와 EventInstance의 연결을 EventRepository가 중앙에서 관리
2. **단순 연결 맵 방식**: `callerId -> instanceId` 형태의 맵을 사용한 연결 관리 (옵션 A 채택)
3. **책임 분리**: EventRepository는 저장소 역할, EventHandler는 실행 관리 역할로 분리
4. **컨텍스트 전달**: EventCaller가 명시적으로 컨텍스트(callerId, eventName, target)를 전달
5. **작동 여부 분리**: bound 속성과 enabled 속성을 분리하여 유연한 상태 제어

### 해결되지 않은 쟁점

1. **Map 키 타입**: Symbol 키와 문자열 키 혼용 문제
2. **WeakMap 적용**: 문자열 키 방식과 WeakMap 권장 간의 모순
3. **재진입 방지**: 구체적인 메커니즘 미정
4. **EventHandler 역할**: 정확한 역할 정의 부재
5. **EventAction 용어**: caller 용어 혼동 문제
