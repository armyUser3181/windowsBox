# CodeBox 진행상황 요약

이 문서는 현재까지의 구현 상태를 빠르게 파악하기 위해 정리한 요약 문서입니다.

## 1. 프로젝트 목표
CodeBox는 이벤트 기반 아키텍처를 활용한 웹 UI 프레임워크를 목표로 하는 프로젝트입니다.

현재는 다음 세 가지 축으로 구성된 기본 구조를 갖추고 있습니다.

- 이벤트 시스템: 이벤트 등록, 실행, 분기 처리
- 윈도우 시스템: 윈도우 UI 컴포넌트 계층
- 디버거/테스터 시스템: 로깅, 성능 측정, 실험용 테스트 코드

## 2. 현재까지 구현된 내용

### 완료된 부분
- 기본 프로젝트 폴더 구조 정리
- 이벤트 관련 클래스 파일 생성
- 윈도우 관련 클래스 파일 생성
- 디버거 클래스 기본 구현
- 테스터 클래스 기본 구현
- CSS 스타일 시트 기본 구성

### 현재 확인되는 핵심 흐름
- 진입점은 index.js에서 시작됨
- index.js는 DebuggerTaster를 불러와 실행함
- 이벤트 처리 흐름은 대략 다음과 같음
  - EventCaller가 이벤트를 감지
  - EventInstance가 실행 제어를 담당
  - EventAction이 실제 로직 수행
  - resolve 함수가 흐름을 마무리

## 3. 현재 구현 상태 상세

### Event 시스템
구조는 어느 정도 정리되어 있지만, 실제 연결 로직은 아직 미완성입니다.

- EventRepository
  - 기본 구조는 존재함
  - bind, unbind, renew 메서드는 아직 TODO 상태

- EventInstance
  - 기본 객체 구조는 존재함
  - bind, unbind는 아직 구현되지 않음
  - run()은 resolve 호출까지 연결되어 있음

- EventCaller
  - 기본 구조는 존재함
  - bind, unbind, call은 아직 구현되지 않음

### Window 시스템
윈도우 UI의 기본 클래스 계층은 정리되어 있습니다.

- cwindow
  - 기본 객체로 생성됨
  - Windowbar와 WindowSizebar를 초기화함

- 하위 버튼/이벤트 클래스들
  - 닫기, 최대화, 최소화 관련 클래스가 존재함

- 다만 실제 창 조작 로직은 아직 충분히 완성되지 않은 상태로 보임

### Debugger 시스템
디버깅 관련 기능은 비교적 잘 정리되어 있습니다.

- log(): 로그 출력
- error(): 에러 로그 출력
- printMemoryUsage(): 메모리 사용량 출력
- measure(): 실행 시간 측정

### Taster 시스템
테스트용 클래스가 존재하며, 디버거와 연결된 테스트 흐름을 확인할 수 있습니다.

## 4. 확인된 미완성 또는 잠재 이슈

- EventRepository의 실제 이벤트 바인딩 로직이 구현되지 않음
- EventCaller에서 실제 이벤트 호출 연결이 구현되지 않음
- EventInstance의 repository 등록/해제 로직이 구현되지 않음
- EventHandler가 아직 비어 있음
- main.js가 비어 있음
- Window.js에서 WindowSizebar를 참조하지만 실제 파일이 없을 가능성이 있음

## 5. 현재 프로젝트의 한 줄 평가
현재 프로젝트는 “구조 설계와 기본 클래스 구성은 완료되었지만, 실제 동작 연결은 아직 진행 중인 상태”입니다.

## 6. 다음 우선순위
1. 이벤트 시스템의 실제 바인딩 연결 구현
2. 윈도우 이벤트 처리 로직 보완
3. 누락된 파일 또는 참조 오류 점검
4. 실제 동작 확인을 위한 테스트 흐름 구성

## 7. 요약
지금 상태는 “기본 프레임워크 뼈대는 갖췄고, 핵심 기능의 실제 연결 작업이 남아 있는 단계”입니다.
