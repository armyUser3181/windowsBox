# CodeBox 진행상황 요약

이 문서는 현재까지의 구현 상태를 빠르게 파악하기 위해 정리한 요약 문서입니다. 상세 내용은 **[COMPREHENSIVE_PROJECT_DOCUMENTATION.md](./COMPREHENSIVE_PROJECT_DOCUMENTATION.md)**를 참조하십시오.

## 1. 프로젝트 목표
CodeBox는 이벤트 기반 아키텍처를 활용한 웹 UI 프레임워크를 목표로 하는 프로젝트입니다.

현재는 다음 세 가지 축으로 구성된 기본 구조를 갖추고 있습니다.

- 이벤트 시스템: 이벤트 등록, 실행, 분기 처리
- 윈도우 시스템: 윈도우 UI 컴포넌트 계층
- 디버거/테스터 시스템: 로깅, 성능 측정, 실험용 테스트 코드

## 2. 현재까지 구현된 내용

### 완료된 부분
- 기본 프로젝트 폴더 구조 정리
- 이벤트 관련 클래스 파일 생성 (스켈레톤)
- 윈도우 관련 클래스 파일 생성 (클래스 껍데기)
- 디버거 클래스 완전 구현
- 테스터 클래스 기본 구현
- CSS 스타일 시트 기본 구성
- Event 시스템 설계 문서 3차례 개정 완료

### 현재 확인되는 핵심 흐름
- 진입점은 index.js에서 시작됨
- index.js는 DebuggerTaster만 실행함 (Window 시스템 미실행)
- 이벤트 처리 흐름은 설계상 다음과 같음
  - EventCaller가 이벤트를 감지
  - EventInstance가 실행 제어를 담당
  - EventAction이 실제 로직 수행
  - resolve 함수가 흐름을 마무리

## 3. 현재 구현 상태 상세

### Event 시스템
현재 스켈레톤 단계입니다. 클래스 구조와 JSDoc은 있지만 실제 로직은 없습니다.

- EventRepository
  - 기본 구조 존재
  - bind, unbind, renew 메서드는 TODO 상태
  - Map 키 타입 불일치 (Symbol 선언 vs id 사용)

- EventInstance
  - 기본 객체 구조 존재
  - bind, unbind는 구현되지 않음
  - run()은 타입 가드만 구현됨

- EventCaller
  - 기본 구조 존재
  - bind, unbind, call은 구현되지 않음

- EventHandler
  - 비어있음

### Window 시스템
현재 클래스 껍데기 단계입니다. DOM 연결이 없습니다.

- cwindow
  - 기본 객체 구조 존재
  - Windowbar와 WindowSizebar 초기화
  - DOM 참조 없음

- 하위 버튼/이벤트 클래스들
  - 닫기, 최대화, 최소화 관련 클래스 존재
  - WindowButtonElement 비어있음
  - WindowSizebar 파일 누락

- 실제 창 조작 로직 미구현

### Debugger 시스템
유일하게 완전히 구현된 모듈입니다.

- log(): 로그 출력
- error(): 에러 로그 출력
- printMemoryUsage(): 메모리 사용량 출력
- measure(): 실행 시간 측정

### Taster 시스템
테스트용 클래스가 존재하며, 디버거와 연결된 테스트 흐름을 확인할 수 있습니다.

## 4. 확인된 미완성 또는 잠재 이슈

### 치명적 이슈
- WindowSizebar.js 파일 누락으로 런타임 에러 위험
- index.js가 cwindow를 import조차 하지 않음
- Window 시스템과 DOM이 완전히 분리됨

### 설계 이슈
- Event 시스템에서 Symbol 키와 문자열 키 혼용
- EventAction에서 caller 용어 혼동 (콜백 함수 vs EventCaller 객체)
- WeakMap 언급과 문자열 키 방식 간 모순
- 재진입 방지 구체적 메커니즘 미정

### 구현 이슈
- Event 시스템 전체가 TODO 스텁 상태
- Window 시스템에 DOM 연결 로직 부재
- EventHandler가 비어있음
- main.js가 비어있음

## 5. 현재 프로젝트의 한 줄 평가
현재 프로젝트는 "설계 문서는 상세히 작성되었고, 클래스 스켈레톤은 존재하지만, 실제 동작 연결은 전혀 이루어지지 않은 상태"입니다.

## 6. 다음 우선순위

### 즉시 실행 (이번 작업 블록)
1. WindowSizebar.js 파일 생성
2. index.js에 cwindow 인스턴스 생성 + DOM 마운트 로직 추가
3. Map 키 타입 통일 (id vs symbol)

### 단기 목표
1. Event 시스템 구현 (3차 설계안 기준)
2. 재진입 방지 로직 구체화
3. Debugger log/error 구분 개선
4. Window 시스템 DOM 연결 구현

## 7. 요약
지금 상태는 "설계 중심 접근으로 상세한 설계 문서가 작성되었고, 클래스 스켈레톤이 존재하지만, 실제 구현과 연결 작업이 전혀 이루어지지 않은 단계"입니다.
